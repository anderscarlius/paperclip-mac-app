# Phase 1

Goal:
- Create a stable baseline for user-environment optimization before larger behavior changes.

Phase 1 checklist:
- Confirm runtime context is visible and trustworthy during local runs.
- Define the first user journey to optimize.
- Capture baseline observations before changes.
- Record each experiment in `experiment-log.md`.
- Request a more detailed next prompt before starting the next step.

Suggested first inputs:
- target user persona
- target workflow
- current friction points
- measurable success criteria
- constraints or non-goals

Expected outputs:
- a short baseline summary
- a prioritized hypothesis list
- a concrete next experiment prompt
