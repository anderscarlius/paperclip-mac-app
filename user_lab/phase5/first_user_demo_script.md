# First User Demo Script

1. Open Paperclip.
2. Show Setup Health.
3. Point out the `Ready to demo` panel.
4. Point out readiness cards.
5. Select or confirm the workspace.
6. Click `Analyze this workspace`.
7. Explain the read-only confirmation.
8. Prepare the request.
9. Collect limited metadata.
10. Show the first workspace summary.
11. Approve the README excerpt step.
12. Show the improved summary.
13. Approve the manifest fields step.
14. Show the more useful summary.
15. Show the first successful run checklist.
16. Point out safety transparency and the private alpha note.
17. Show the local-only feedback questions.
18. Explain what is not built yet.
19. Ask the tester for feedback.

## Suggested tester feedback questions

- Did you understand what Paperclip inspected?
- Did the safety copy feel clear?
- Did you trust the README read step?
- Did you trust the manifest-field read step?
- Was the first summary useful?
- Did the first successful run checklist match what you thought success meant?
- Did the Ready to demo panel match what you thought “ready” meant?
- What would you expect the next button to do?
- Would you try this on your own project?
- Did the feedback questions help you explain what felt clear or unclear?
