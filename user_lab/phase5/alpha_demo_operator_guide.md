# Paperclip Private Alpha Demo Operator Guide

## Purpose

Help an operator run a calm first-user walkthrough.

## Before demo

- confirm repo is clean
- confirm latest commit is pushed
- run focused tests
- open Paperclip
- check Startup status
- check Ready to demo panel

## During demo

- explain private alpha scope
- show Setup Health
- show Ready to demo
- run Analyze Workspace
- collect limited metadata
- show first summary
- optionally read README
- optionally read manifest fields
- review what was inspected and not inspected
- ask feedback questions

## What not to claim

- no AI analysis yet
- no code editing
- no command execution
- no dependency health
- no security review
- no production readiness

## After demo

- record feedback
- record confusion points
- record desired next action
