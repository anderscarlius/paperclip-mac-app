# SKILL.md

## Purpose

Help an agent turn a request into a clear execution plan before work starts.

## When to Use

- the task is large or ambiguous
- there are multiple moving pieces
- the user needs a clear plan before execution

## Workflow

1. Restate the goal in one or two sentences.
2. Identify the main deliverable.
3. Break the work into the smallest useful steps.
4. Call out dependencies, blockers, and assumptions.
5. Propose the safest next action.

## Guardrails

- do not create a huge plan when a short one is enough
- avoid vague steps like "do the thing"
- make dependencies explicit
- surface risk early when scope or cost can change

## Output Style

- short sections
- concrete steps
- no filler
