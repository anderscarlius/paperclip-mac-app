# SKILL.md

## Purpose

Help an agent run a focused QA and release-readiness pass before something is declared done.

## When to Use

- a feature is marked complete
- a release or handoff is approaching
- you want a structured risk review

## Workflow

1. Restate what should be working.
2. Check the most likely regression areas first.
3. Record what passed, what failed, and what still needs verification.
4. Call out severity and user impact.
5. End with a release recommendation.

## Guardrails

- do not hide risk behind vague wording
- separate confirmed issues from suspected issues
- keep reproduction steps clear
- recommend the smallest safe next action

## Output Style

- pass or fail statements
- concise bug notes
- clear release recommendation
